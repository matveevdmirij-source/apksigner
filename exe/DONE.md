## ⚙️ Что происходит "под капотом"?

### Процесс подписи APK:

```
APK файл
  ↓
1. Анализ структуры
  ↓
2. Создание/использование Keystore
  ↓
3. Подпись через jarsigner (RSA 2048-bit, SHA1)
  ↓
4. Экспорт сертификата (PEM формат)
  ↓
5. Вычисление fingerprints (SHA256, SHA1)
  ↓
6. Генерация инструкций для маркета
  ↓
✅ app-signed.apk готов!
```

### Процесс подписи AAB:

```
AAB файл
  ↓
1. Анализ структуры (как APK)
  ↓
2. Использование ТОГО ЖЕ Keystore ✅
  ↓
3. Подпись через jarsigner (тот же алгоритм)
  ↓
4-6. Те же шаги что и для APK
  ↓
✅ app-signed.aab готов!
   (без RuStore инструкций - только Google Play)
```

## 🔐 Безопасность и Best Practices

### ✅ ПРАВИЛА (НЕ НАРУШАЙТЕ!):

1. **ОДИН Keystore на приложение!**
   ```
   ✅ Правильно: 
      Keystore A → app v1.0 APK + app v1.0 AAB
   
   ❌ Неправильно:
      app v1.0 APK → Keystore A
      app v1.0 AAB → Keystore B (разные!)
   ```

2. **Защищайте Keystore:**
   ```
   ✅ Сохраните: app-release-key.jks в безопасном месте
   ✅ Запомните: пароль (android_key_password)
   ✅ Сделайте: от резервную копию
   ❌ НЕ выкладывайте: на GitHub или интернет
   ```

3. **Используйте один Keystore везде:**
   ```
   ✅ APK на Google Play
   ✅ APK на RuStore
   ✅ AAB на Google Play
   
   Все с ОДНИМ Keystore!
   ```

## 📊 Сравнение форматов

| Параметр | APK | AAB |
|----------|-----|-----|
| **Размер** | 40-100 MB | 10-30 MB |
| **Google Play** | ✅ Да | ✅ Да (рекомендуется) |
| **RuStore** | ✅ Да | ❌ Нет |
| **Установка** | Прямая | Только через маркет |
| **Оптимизация** | Нет | Автоматическая |
| **Beta тестирование** | ✅ Да | ✅ Да |
| **Локальное тестирование** | ✅ Да | ⚠️ Нужен bundletool |
