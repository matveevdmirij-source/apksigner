# APK Signer для Play Market и RuStore (v1.2+)

Приложение с графическим интерфейсом для генерации PEM ключей и цифровых подписей **APK и AAB файлов**, предназначенных для загрузки на Google Play Market и RuStore.

## 🎯 Основные функции

- ✅ Загрузка **APK и AAB файлов** (новое!)
- ✅ Выбор формата в графическом интерфейсе
- ✅ Создание нового Keystore или использование существующего
- ✅ Подпись APK и AAB файлов
- ✅ Экспорт сертификата в PEM формат
- ✅ Генерация подписей для Google Play Market
- ✅ Генерация подписей для RuStore (только APK)
- ✅ Вычисление SHA256 и SHA1 отпечатков сертификатов
- ✅ Сохранение всех необходимых файлов

## 📚 Что такое AAB?

**AAB (Android App Bundle)** - новый формат для Google Play Store со следующими преимуществами:
- **Размер:** На 40% меньше чем APK
- **Оптимизация:** Автоматическая генерация для каждого устройства
- **Скорость:** Пользователи скачивают только нужные компоненты

Для полной информации смотрите: **[AAB_SUPPORT.md](AAB_SUPPORT.md)**

### 📱 Рабочий процесс

1. **Выбрать APK файл**
   - Нажмите кнопку "Обзор" рядом с полем "Выбор APK файла"
   - Выберите ваш APK файл

2. **Выбрать Keystore (опционально)**
   - Если у вас есть существующий keystore, нажмите "Обзор"

   - Нажмите "Обзор" рядом с "Папка для сохранения файлов"

4. **Нажать "Генерировать ключи и подписи"**
   - Программа начнет процесс генерации
   - Все шаги будут отображены в окне информации

5. **Получить результаты**
   - Все файлы будут сохранены в выбранной папке

## 📁 Выходные файлы

### Основные файлы:
- `app-release-key.jks` - Keystore с приватным ключом
- `app-signed.apk` - Подписанный APK файл
- `certificate.pem` - Публичный сертификат в PEM формате
- `certificate.der` - Публичный сертификат в DER формате

### Информационные файлы:
- `playmarket_certificate.pem` - Сертификат для Google Play
- `playmarket_fingerprints.txt` - Отпечатки для Play Market
- `rustore_signature_info.txt` - Информация для RuStore
- `app-release-key.jks.info` - Информация о keystore в JSON

## 🔑 Параметры по умолчанию

- **Пароль Keystore:** `android_key_password`
- **Пароль Ключа:** `android_key_password`
- **Alias Ключа:** `release_key`
- **Размер ключа:** 2048 бит (RSA)
- **Срок действия:** 10000 дней (~27 лет)

## 📤 Загрузка на Google Play Market

1. Используйте файл `app-signed.apk` для загрузки
2. Нужно будет укзать PEM сертификат из папки
3. В Google Play Console перейдите в настройки приложения
4. Укажите SHA256 отпечаток из `playmarket_fingerprints.txt`

## 📤 Загрузка на RuStore

1. Используйте файл `app-signed.apk` для загрузки
2. Загрузите файл `certificate.pem` как публичный сертификат
3. Убедитесь, что SHA256 отпечатки совпадают
4. Следуйте инструкциям в файле `rustore_signature_info.txt`

## ⚙️ Решение проблем

### 1️⃣ Ошибка: "keytool не найден"

**Описание:** Программа не может найти инструмент keytool для создания keystore.

**Причины:**
- JDK установлен неправильно
- keytool не добавлен в переменную PATH
- Установлена только JRE вместо JDK
- Путь к JDK содержит пробелы или спецсимволы

**Решение:**
```bash
# 1. Проверить наличие keytool:
keytool -help

# 2. Если ошибка - добавить JDK в PATH:
# На Windows - переменная окружения JAVA_HOME должна быть:
# C:\Program Files\Java\jdk-11.0.17

# 3. Переустановить JDK с опцией "Add to PATH"

# 4. Перезагрузить компьютер после установки
```

### 2️⃣ Ошибка: "Keystore was tampered with, or password was incorrect"

**Описание:** При подписи программа не может открыть файл keystore.

**Причины:**
- Неправильный пароль keystore
- Файл keystore повреждён
- Файл keystore не является валидным ZipFile
- Файл был изменён или скопирован неправильно
- Антивирус повредил файл

**Решение:**
```bash
# 1. Проверить целостность keystore:
keytool -list -keystore app-release-key.jks -storepass android_key_password

# 2. Если не работает:
# - Удалить старый keystore (app-release-key.jks)
# - Программа создаст новый при следующем запуске
# - Убедиться, что пустой пароль "android_key_password"

# 3. Проверить права доступа к файлу:
# Правый клик → Свойства → Безопасность
```

### 3️⃣ Ошибка: "APK file not found"

**Описание:** Программа не может найти APK файл по указанному пути.

**Причины:**
- Неправильный путь к файлу
- APK находится в папке с кириллицей (русские буквы)
- APK файл был удалён или переименован
- Недостаточно прав на чтение файла
- Путь содержит спецсимволы (!, @, #, и т.д.)

**Решение:**
```bash
# 1. Убедиться, что файл существует:
dir C:\path\to\app.apk

# 2. Переименовать APK без спецсимволов:
# Неправильно: app-релиз.apk, app@1.apk, app!.apk
# Правильно:  app-release.apk, app-v1.apk, app_final.apk

# 3. Скопировать APK в папку программы и выбрать его

# 4. Использовать абсолютный путь без пробелов:
# Хорошо:  C:\Users\Admin\Desktop\exe\app.apk
# Плохо:   C:\Users\Admin\My Documents\app.apk
```

### 4️⃣ Ошибка: "Invalid keystore format"

**Описание:** Файл keystore имеет неправильный формат или повреждён.

**Причины:**
- Файл не является JKS (Java KeyStore)
- Файл повреждён или неполный
- Был выбран неправильный файл
- Файл скачан из интернета и поврежден при передаче
- Неправильная кодировка имени файла

**Решение:**
```bash
# 1. Проверить формат keystore:
keytool -list -keystore file.jks -storepass android_key_password

# 2. Если команда выдаёт ошибку:
# - Удалить файл keystore
# - Создать новый (программа создаст автоматически)

# 3. Убедиться расширение файла .jks или .keystore

# 4. Если был скачан keystore:
# - Проверить целостность (MD5, SHA256 хеш)
# - Скачать заново
```

### 5️⃣ Ошибка: "jarsigner: unable to sign jar"

**Описание:** Ошибка при попытке подписать APK файл.

**Причины:**
- APK файл повреждён или не соответствует формату JAR/ZIP
- Неправильный alias ключа (не "release_key")
- Ключ с таким alias не существует в keystore
- Файл APK заблокирован другим процессом
- Недостаточно свободного места на диске

**Решение:**
```bash
# 1. Проверить, что это валидный ZIP/JAR (APK):
# Открыть APK архиватором (7-Zip, WinRAR, или встроенный архиватор)
# Если не открывается - APK повреждён

# 2. Проверить наличие ключа в keystore:
keytool -list -v -keystore app-release-key.jks -storepass android_key_password

# 3. Перезаписать APK файл (скопировать заново из исходника)

# 4. Закрыть все архиваторы и IDE

# 5. Проверить место на диске:
# Windows: Щелчок правой кнопкой на диск C: → Свойства
```

### 6️⃣ Ошибка: "Certificate validity period has ended"

**Описание:** Сертификат в keystore истёк и больше не действителен.

**Причины:**
- Keystore создан очень давно (более 27 лет назад - это маловероятно в 2026 году)
- Дата на компьютере установлена неправильно (например, год 1980)
- Срок действия был установлен на малое значение при создании

**Решение:**
```bash
# 1. Проверить текущую дату на компьютере:
echo %date%

# 2. Если дата неправильная - установить правильную:
# Windows: Параметры → Время и тип → Дата и время → Установить время

# 3. Если дата правильная:
# - Удалить старый keystore
# - Создать новый (срок 10000 дней = ~27 лет)

# 4. Проверить срок действия:
keytool -list -v -keystore app-release-key.jks -storepass android_key_password
```

### 7️⃣ Ошибка: "PermissionError" / "Permission denied"

**Описание:** Отсутствуют права доступа на чтение или запись файлов.

**Причины:**
- Файл или папка находится в защищённой директории
- Неправильные права доступа на файлы
- Файл используется другим приложением (IDE, архиватор)
- Антивирус блокирует доступ к файлам
- Файл отмечен как "только для чтения"

**Решение:**
```bash
# 1. Запустить программу с правами администратора

# 2. Переместить файлы в папку с полными правами:
# Хорошо:   C:\Users\Admin\Desktop\exe\
# Плохо:    C:\Program Files\...
#           C:\Windows\System32\...
#           C:\ProgramData\...

# 3. Проверить свойства файла:
# Правый клик → Свойства → убрать галочку "Только для чтения"

# 4. Проверить права доступа:
# Правый клик на файл → Свойства → Безопасность

# 5. Закрыть архивные менеджеры, IDE и другие приложения

# 6. Отключить антивирус временно (только для тестирования)

# 7. Если всё ещё не работает:
# - Переименовать файл
# - Скопировать в другую папку
# - Перезагрузить компьютер
```

### 8️⃣ Ошибка: "UnicodeDecodeError" или проблема с кодировкой

**Описание:** Проблема с кодировкой символов при работе с путями.

**Причины:**
- Путь к файлам содержит русские символы (кириллица)
- Неправильная кодировка системы
- Конфликт между UTF-8 и Windows-1251

**Решение:**
```bash
# 1. Использовать пути БЕЗ русских букв:
# Правильно:  C:\Users\Admin\Desktop\exe\app.apk
# Неправильно: C:\Пользователи\Админ\Рабочий стол\app.apk

# 2. Переименовать папки на английский язык:
# Рабочий стол → Desktop
# Мои документы → My Documents

# 3. Переместить всё в простую папку:
# C:\signed_apps (лучше всего)

# 4. Установить кодировку UTF-8 для консоли:
# chcp 65001  (Windows)
```

### 9️⃣ Ошибка: "Private key password is incorrect"

**Описание:** При подписи указан неправильный пароль для приватного ключа.

**Причины:**
- Пароль ключа отличается от пароля keystore
- Опечатка в пароле
- Пароль был изменен при создании keystore
- Пароль содержит спецсимволы, которые не были правильно переданы

**Решение:**
```bash
# При создании keystore используются одинаковые пароли:
# Пароль keystore: android_key_password
# Пароль ключа:    android_key_password

# Если забыли пароль - создать новый keystore
```

### 🔟 Ошибка: "Out of memory" / Memory error

**Описание:** Недостаточно оперативной памяти для подписи большого APK.

**Причины:**
- APK файл очень большой (более 500 MB)
- На компьютере недостаточно свободной памяти
- Другие приложения занимают много памяти
- Утечка памяти в программе

**Решение:**
```bash
# 1. Закрыть ненужные приложения (браузеры, IDE, и т.д.)

# 2. Проверить свободную память:
# Windows: Ctrl+Shift+Esc → Память

# 3. Попробовать подписать APK на другом компьютере с большей памятью

# 4. Проверить, не поврежден ли APK файл
```

## ✅ Как избежать ошибок (чек-лист)

- ✅ Использовать простые пути без спецсимволов и кириллицы
- ✅ Сохранить keystore в безопасном месте с резервной копией
- ✅ Запомнить пароль (android_key_password)
- ✅ Проверить APK перед подписью (открыть архиватором)
- ✅ Запустить программу от администратора
- ✅ Мониторить лог на предмет ошибок
- ✅ Создать keystore только один раз на приложение
- ✅ Никогда не стирать файл keystore!
- ✅ Проверить достаточно ли свободной памяти
- ✅ Закрыть все другие приложения

## 🔒 Безопасность

⚠️ **ВАЖНО:**
- Никогда не делитесь файлом `app-release-key.jks`
- Храните `app-release-key.jks` в безопасном месте
- Сохраните пароли от keystore в безопасном месте
- Не загружайте `app-release-key.jks` в публичные хранилища (GitHub, GitLab)

## 💡 Советы

1. **Создавайте резервные копии**
   - Сохраняйте keystore в безопасном месте
   - Без него не сможете обновлять приложение

2. **Используйте один keystore**
   - Для каждого приложения нужен свой keystore
   - Один keystore используется для всех версий приложения

3. **Проверяйте отпечатки**
   - SHA256 отпечатки должны совпадать в Play Market и RuStore
   - Это гарантирует подлинность приложения
