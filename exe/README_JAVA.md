# APK Signer - Java версия (GUI) v1.2+

Полнофункциональное приложение для создания подписей **APK и AAB файлов** на **Java** с графическим интерфейсом (Swing).

## ✨ Возможности

✅ Поддержка **APK и AAB форматов** (новое!)  
✅ Выбор формата в графическом интерфейсе  
✅ Создание нового keystore с приватным ключом  
✅ Подпись APK/AAB файла с помощью jarsigner  
✅ Экспорт публичного сертификата в PEM формат  
✅ Генерация подписей для Google Play Market  
✅ Генерация подписей для RuStore (только APK)  
✅ Вычисление SHA256 и SHA1 отпечатков  
✅ Графический интерфейс (Swing)  
✅ Прогресс-бар и лог операций в реальном времени  

## 📋 Требования

- **Java Development Kit (JDK) 8 или выше** ✅ (уже установлена!)
- Windows, macOS или Linux

> Java 11 уже установлена на вашей системе, поэтому приложение готово к использованию!

### Шаг 2: Выбрать APK файл
1. Нажмите кнопку **"Обзор"** рядом с "Выбор APK файла"
2. Выберите ваш APK файл (например: `app-release.apk`)
3. Файл будет проанализирован

### Шаг 3: Выбрать Keystore (опционально)
- Если у вас уже есть keystore файл (`.jks`), нажмите "Обзор" и выберите его
- Если оставить пусто, будет **создан новый keystore**

### Шаг 4: Выбрать папку для сохранения
1. Нажмите кнопку **"Обзор"** рядом с "Папка для сохранения файлов"
2. Выберите или создайте папку (например: `C:\signed_apps`)

### Шаг 5: Запустить генерацию
1. Нажмите кнопку **"Генерировать ключи и подписи"**
2. Смотрите прогресс в окне лога
3. Дождитесь завершения операции

## 🔴 Ошибки при создании подписи APK

### 1️⃣ Ошибка: "jarsigner not found"

**Описание:** Программа не может найти инструмент jarsigner для подписи APK.

**Причины:**
- JDK установлен неправильно
- jarsigner не добавлен в переменную PATH
- Установлена только JRE вместо JDK

**Решение:**
```bash
# Проверить наличие jarsigner
jarsigner -help

# Если ошибка - добавить JDK в PATH:
# Переменная окружения JAVA_HOME должна указывать на папку JDK
# Например: C:\Program Files\Java\jdk-11.0.17
```

### 2️⃣ Ошибка: "Keystore was tampered with, or password was incorrect"

**Описание:** При подписи APK программа не может открыть файл keystore.

**Причины:**
- Неправильный пароль keystore
- Файл keystore повреждён
- Файл keystore не является валидным ZipFile
- Файл был изменён во время работы

**Решение:**
```bash
# Проверить целостность keystore:
keytool -list -keystore app-release-key.jks -storepass android_key_password

# Если не работает:
# 1. Удалить старый keystore и создать новый
# 2. Убедиться, что никто не изменял файл
# 3. Проверить права доступа к файлу
```

### 3️⃣ Ошибка: "APK file not found"

**Описание:** Программа не может найти APK файл по указанному пути.

**Причины:**
- Неправильный путь к файлу
- Путь содержит кириллицу или специальные символы
- APK файл был удалён или переименован
- Недостаточно прав на чтение файла

**Решение:**
```bash
# 1. Проверить, существует ли файл:
dir C:\path\to\app.apk

# 2. Переименовать APK без спецсимволов:
# Неправильно: app-релиз.apk
# Правильно:  app-release.apk

# 3. Скопировать APK в папку программы
# 4. Удалить пробелы из пути
```

### 4️⃣ Ошибка: "Invalid keystore format"

**Описание:** Файл keystore имеет неправильный формат.

**Причины:**
- Файл не является JKS (Java KeyStore)
- Файл повреждён или неполный
- Выбран неправильный файл (не keystore)
- Файл был загружен по сети и поврежден

**Решение:**
```bash
# 1. Проверить формат keystore:
keytool -list -keystore file.jks -storepass password

# 2. Удалить повреждённый keystore и создать новый

# 3. Убедиться, что это действительно файл keystore
# Правильные расширения: .jks, .keystore
```

### 5️⃣ Ошибка: "jarsigner: unable to sign jar"

**Описание:** Ошибка при попытке подписать APK файл.

**Причины:**
- APK файл повреждён и не соответствует формату ZIP
- Неправильный alias ключа
- Ключ с таким alias не существует в keystore
- Файл заблокирован другим процессом

**Решение:**
```bash
# 1. Проверить, что это валидный ZIP (APK):
# Попробовать открыть APK архиватором (7-Zip, WinRAR)

# 2. Проверить наличие ключа в keystore:
keytool -list -keystore app-release-key.jks -storepass android_key_password

# 3. Убедиться, что никакой процесс не блокирует файлы

# 4. Переместить базовые файлы в папку без пробелов:
# C:\signed_apps (хорошо)
# C:\Program Files\Apps\signed (плохо - пробелы)
```

### 6️⃣ Ошибка: "Certificate validity period has ended"

**Описание:** Сертификат в keystore истёк и больше не действителен.

**Причины:**
- Keystore создан очень давно (более 27 лет назад)
- Дата на компьютере установлена неправильно
- Срок действия сертификата был установлен на малое значение

**Решение:**
```bash
# 1. Проверить дату на компьютере
date  (Linux/Mac)
date /T  (Windows)

# 2. Создать новый keystore с большим сроком действия:
# При создании используется: -validity 10000 (27 лет)

# 3. Если keystore старый и истёк - создать новый:
# Удалить app-release-key.jks
# Запустить программу заново, она создаст новый keystore
```

### 7️⃣ Ошибка: "Only a keystore can be imported into a keystore"

**Описание:** При импорте ошибка параметров keystore.

**Причины:**
- Неправильный тип файла
- Параметры команды keytool неправильные
- Конфликт версий

**Решение:**
```bash
# Проверить синтаксис команды keytool:
keytool -help

# Убедиться, что используются правильные параметры для генерации:
keytool -genkey -v -keystore keystore.jks -keyalg RSA -keysize 2048
```

### 8️⃣ Ошибка: "Unable to access keystore: windows-1251 encoding"

**Описание:** Проблема с кодировкой при работе с путями на кириллице.

**Причины:**
- Путь к файлам содержит русские символы
- Неправильная кодировка командной строки
- Проблема с Windows кодировкой

**Решение:**
```bash
# 1. Использовать пути без кириллицы:
# Правильно:  C:\Users\Admin\Desktop\exe\app.apk
# Неправильно: C:\Пользователи\Админ\Рабочий стол\app.apk

# 2. Переместить файлы в папку с латинскими символами

# 3. Переименовать папки на английский
```

### 9️⃣ Ошибка: "Private key password is incorrect"

**Описание:** При подписи указан неправильный пароль для приватного ключа.

**Причины:**
- Пароль ключа отличается от пароля keystore
- Опечатка в пароле
- Пароль содержит спецсимволы

**Решение:**
```bash
# 1. Проверить пароль ключа в keystore:
keytool -list -v -keystore app-release-key.jks -storepass android_key_password

# 2. При создании keystore используется одинаковый пароль для обоих
# Пароль keystore: android_key_password
# Пароль ключа:    android_key_password

# 3. Если забыли пароль - создать новый keystore
```

### 🔟 Ошибка: "Out of memory" / Java heap space

**Описание:** Недостаточно оперативной памяти для подписи большого APK.

**Причины:**
- APK файл очень большой (более 500 MB)
- На компьютере недостаточно свободной памяти
- Java использует ограниченный heap размер

**Решение:**
```bash
# 1. Увеличить heap размер Java:
java -Xmx1024m APKSigner

# 2. Закрыть ненужные приложения

# 3. Проверить, сколько памяти доступно:
# Windows: Ctrl+Shift+Esc (диспетчер задач) → Производительность
```

### 1️⃣1️⃣ Ошибка: "Permission denied" / "Access is denied"

**Описание:** Отсутствуют права доступа на чтение или запись файлов.

**Причины:**
- Файл находится в защищённой папке
- Неправильные права доступа к файлам
- Файл используется другим приложением
- Антивирус блокирует доступ

**Решение:**
```bash
# 1. Запустить программу от администратора:
# Правый клик на run_java.bat → "Запустить от администратора"

# 2. Переместить файлы в папку с полными правами:
# Хорошо:   C:\Users\Admin\Desktop\exe\
# Плохо:    C:\Program Files\...
#           C:\Windows\System32\...

# 3. Проверить права доступа:
# Правый клик на папку → Свойства → Безопасность

# 4. Отключить антивирус временно при тестировании
```

### 1️⃣2️⃣ Ошибка: "Invalid signature file digest for Manifest main attributes"

**Описание:** При верификации подписи найдены ошибки целостности APK.

**Причины:**
- APK был изменён после подписи
- Файлы внутри APK повреждены
- Неправильная подпись применена

**Решение:**
```bash
# 1. Проверить целостность подписанного APK:
jarsigner -verify -verbose app-signed.apk

# 2. Если ошибка - переподписать APK заново

# 3. Убедиться, что не прикасались к файлу после подписи
```

## ✅ Как избежать ошибок

1. **Используйте простые пути** без спецсимволов и кириллицы
2. **Сохраняйте keystore** в безопасном месте с резервной копией
3. **Помните пароли** (обычно: android_key_password)
4. **Проверяйте APK** перед подписью (откройте архиватором)
5. **Запускайте от администратора** для избежания проблем с правами
6. **Мониторьте лог** на предмет ошибок
7. **Создавайте новый keystore** только один раз на приложение
8. **Никогда не теряйте keystore** файл!
