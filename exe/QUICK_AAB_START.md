# ⚡ Быстрый старт с AAB (Android App Bundle)

**Тв хочешь загрузить обновление на Google Play? AAB поможет!**

---

## 🎯 За 5 минут до AAB подписи

### Шаг 1️⃣: Создай AAB файл в Android Studio

```
Android Studio
  → Build (меню)
    → Build Bundle(s) / APK(s)
      → Build Bundle(s)
```

Файл сохранится в папке:
```
YourProject/app/release/app-release.aab
```

<strong>✅ Готова!</strong> Теперь у тебя есть AAB файл.

---

### Шаг 2️⃣: Запусти APK Signer

**Вариант А:** Двойной клик
```
run_java.bat   (базовая версия)
или
run_advanced.bat  (с пересборкой)
```

**Вариант Б:** Командная строка
```bash
java APKSigner
```

<strong>✅ Окно приложения открыто!</strong>

---

### Шаг 3️⃣: Выбери AAB формат

В окне приложения:
1. Найди выпадающий список "Формат"
2. Выбери **AAB** (вместо APK)
3. Прочитай описание AAB

**ВАЖНО:** Это самый главный шаг - если забудешь выбрать AAB, подпишется неправильно!

<strong>✅ AAB формат выбран!</strong>

---

### Шаг 4️⃣: Выбери AAB файл

1. Нажми кнопку "Обзор" (рядом с "Выбор файла")
2. Найди свой AAB файл:
   ```
   YourProject/app/release/app-release.aab
   ```
3. Нажми "Открыть"

**Подсказка:** Диалог будет показывать только `*.aab` файлы (если выбран AAB формат)

<strong>✅ AAB файл выбран!</strong>

---

### Шаг 5️⃣: Выбери папку для сохранения

1. Нажми "Обзор" рядом с "Папка для сохранения файлов"
2. Выбери удобную папку (например `C:\signed_apps`)
3. Нажми "Выбрать папку"

<strong>✅ Папка выбрана!</strong>

---

### Шаг 6️⃣: Подпиши!

1. Нажми большую кнопку **"Генерировать ключи и подписи"**
2. Жди... (обычно 10-30 секунд)
3. Смотри логи в окне программы
4. Когда закончится - нажми кнопку "Обзор" результатов

<strong>✅ AAB подписан!</strong>

---

## 📁 Что получится?

В папке `C:\signed_apps` найдешь:

```
✅ app-signed.aab              ← Это главный файл для Google Play!
✅ certificate.pem             ← Публичный сертификат
✅ app-release-key.jks         ← Приватный ключ (ХРАНИ В БЕЗОПАСНОСТИ!)
✅ playmarket_info.txt         ← Инструкции для Google Play
```

---

## 🚀 Загрузи на Google Play Store

1. Откой **Google Play Console**
   ```
   https://play.google.com/console
   ```

2. Выбери свое приложение

3. Нажми **"Release"** или **"Production"**

4. Нажми **"Create release"**

5. Нажми кнопку загрузки (`app-signed.aab`)

6. Выбери файл:
   ```
   C:\signed_apps\app-signed.aab
   ```

7. Система покажет информацию:
   - Версия должна быть выше старой
   - Сертификат совпадает ✓

8. Нажми **"Review"** → **"Publish"**

<strong>✅ AAB загружен на Google Play!</strong>

## 📊 APK vs AAB - когда что использовать?

| Ситуация | Используй | Причина |
|----------|-----------|---------|
| Google Play Store | AAB | Меньше размер, лучше метрики |
| RuStore | APK | RuStore не поддерживает AAB |
| Beta тестирование | APK | Проще раздавать |
| Локальное тестирование | APK | AAB нужен bundletool для установки |
| Обновление версии | Один формат | Не смешивай! |

## 🆘 Что если что-то пошло не так?

### Ошибка: "AAB файл не найден"
```
→ Проверь что выбран формат AAB (пункт Шаг 3️⃣)
→ Диалог должен показывать только .aab файлы
→ Может быть файл называется не .aab, а .zip?
```

### Ошибка: "java не найден"
```
→ Java должна быть установлена на компьютере
→ Проверь: java -version
→ Если не работает - смотри INSTALL.md
```

### Ошибка при загрузке на Play Market
```
→ Проверь что версия выше чем в старом APK
→ Проверь что сертификат совпадает
→ Смотри playmarket_info.txt с инструкциями
```
