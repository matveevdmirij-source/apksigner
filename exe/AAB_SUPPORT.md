## 🆕 Что такое AAB?

**AAB (Android App Bundle)** - это новый формат распределения приложений, который используется **Google Play Store**.

### Как бы AAB отличается от APK?

| Параметр | APK | AAB |
|----------|-----|-----|
| **Использование** | Прямая установка на устройства | Google Play Store только |
| **Размер** | 40-100+ MB | 10-30 MB (сжимается) |
| **Поддержка устройств** | Все (или указанные в build) | Автоматическая оптимизация |
| **Создание файлов** | 1 APK файл | +1 AAB файл + динамические модули |
| **Загрузка пользователем** | Весь APK | Только нужные компоненты |
| **Маркеты** | Везде | Google Play Store, RuStore (нет) |

---

## 🎯 Когда использовать AAB vs APK?

### ✅ Используйте APK если:
```
- Загружаете на RuStore (не поддерживает AAB)
- Используете beta-тестирование через Google Play
- Нужна мгновенная установка на девайсы
- Тестируете на эмуляторе вручную
```

### ✅ Используйте AAB если:
```
- Загружаете Release версию на Google Play Store
- Хотите оптимизировать размер скачивания
- Нужна автоматическая генерация для разных устройств
- Используете Google Play Console для распределения
```

## 🚀 Как использовать AAB в приложении

### Вариант 1: Basic версия (APKSigner v1.2+)

```
1. Запустить: run_java.bat
2. Выбрать формат: AAB (вместо APK)
3. Выбрать файл: *.aab файл
4. Нажать: "Генерировать ключи и подписи"
5. Результат: app-signed.aab + certificate.pem
```

### Вариант 2: Advanced версия

```
1. Запустить: run_advanced.bat
2. На вкладке "Подпись APK/AAB":
   - Выбрать формат: AAB
   - Выбрать файл: *.aab
   - Нажать: "Генерировать"
3. На вкладке "Пересборка Release":
   - Выбрать проект
   - Нажать: "Пересобрать"
   - Система создаст Release APK (не AAB)
```

## 📋 Что происходит при подписи AAB?

### Процесс для AAB:

```
1. ✓ Анализ AAB файла
   └─ Проверка размера и структуры
   
2. ✓ Создание/использование Keystore
   └─ Сохранение ключей (RSA 2048-bit)
   
3. ✓ Подпись AAB через jarsigner
   └─ Алгоритм: SHA1withRSA
   └─ Результат: app-signed.aab
   
4. ✓ Экспорт сертификата
   └─ Формат: PEM (X.509)
   └─ Результат: certificate.pem
   
5. ✓ Генерация информации для Google Play
   └─ Инструкции загрузки
   └─ Информация о SHA256 fingerprint
```

### Файлы на выходе:

```
Для AAB:
├── app-signed.aab              ← Подписанный AAB (для Google Play)
├── certificate.pem             ← Публичный сертификат
├── playmarket_info.txt         ← Инструкции для Google Play
├── app-release-key.jks         ← Keystore с приватным ключом
└── rustore_info.txt            ← (не может быть использован для RuStore)
```

## 🔐 Безопасность при работе с AAB

### Важные правила:

⚠️ **КРИТИЧНО - то же как для APK:**
1. **ОДИН Keystore на версию приложения**
   - Используйте один и тот же keystore при обновлении
   - Не создавайте новый для каждого релиза!

2. **Защита приватного ключа**
   - Сохраняйте keystore в безопасном месте
   - Никогда не публикуйте app-release-key.jks
   - Помните пароль: android_key_password

3. **Сертификат можно делиться**
   - certificate.pem можно выложить в репозиторий
   - Используется только для проверки целостности

## 📊 Сравнение подписи APK vs AAB

### Одинаковое:
```
✓ Используется один Keystore
✓ Алгоритм подписи: SHA1withRSA
✓ Размер ключа: RSA 2048-bit
✓ Валидность: 10,000 дней (~27 лет)
✓ Инструмент: jarsigner (встроен в Java)
```

### Отличается процесс загрузки:

**Для APK:**
```
Google Play Console
  → Release
    → Загрузить app-signed.apk
    → Система создает для разных устройств
```

**Для AAB:**
```
Google Play Console
  → Release → Production
    → Загрузить app-signed.aab
    → Google Play автоматически создает APK для каждого устройства
    → Пользователи скачивают только нужное им
```

## 🔧 Техническими сведения

### Структура AAB файла:

AAB содержит:
```
app.aab (ZIP архив)
├── BundleConfig.pb          ← Конфигурация пакета
├── base/                    ← Основной модуль
│   ├── manifest/
│   ├── dex/
│   ├── res/
│   ├── lib/
│   └── assets/
├── feature_module_1/        ← Динамический модуль
├── feature_module_2/        ← Динамический модуль
└── instants/                ← Android Instant Apps
```

### Как Google Play использует AAB:

```
1. Загрузка AAB на Google Play
2. Парсинг структуры
3. Создание варианта для каждого устройства:
   - device1 (arm64, 1080x1920) → APK 15MB
   - device2 (armv7, 720x1280) → APK 12MB
   - device3 (x86, 1440x2560) → APK 20MB
4. Пользователь скачивает свой вариант
```

## 🚀 Как начать использовать AAB

### Шаг 1: Обновить приложение

В вашем `build.gradle (app)`:
```gradle
android {
    compileSdkVersion 34
    
    defaultConfig {
        applicationId "com.example.myapp"
        minSdkVersion 21
        targetSdkVersion 34
        versionCode 1
        versionName "1.0"
    }
    
    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

### Шаг 2: Собрать AAB в Android Studio

```
Build → Build Bundle(s) / APK(s) → Build Bundle(s)
→ Сохраняется в: app/release/app-release.aab
```

### Шаг 3: Подписать через наше приложение

```
1. Запустить APKSigner v1.2+
2. Выбрать формат: AAB
3. Выбрать: app-release.aab
4. Нажать: "Подписать"
5. Получить: app-signed.aab
```

### Шаг 4: Загрузить на Google Play

```
Google Play Console
  → Release Management
    → Production
      → Create release
        → Upload app-signed.aab
        → Review → Publish
```

## 📞 Частые вопросы об AAB

**Q: Если я загрузил AAB, могу ли я потом загрузить APK?**
A: Нет! После первого AAB нужно всегда использовать AAB. Для RuStore создавайте отдельный список с APK.

**Q: Мне нужны два keystore для AAB и APK?**
A: Нет! Один keystore на версию приложения для обоих форматов.

**Q: Может ли пользователь установить из AAB?**
A: Нет, только через Google Play. Для тестирования нужен APK (bundletool конвертирует).

**Q: Как тестировать AAB локально?**
A: Использовать bundletool (отдельный инструмент):
```bash
bundletool build-apks --bundle=app-release.aab --output=app.apks --ks=key.jks
bundletool install-apks --apks=app.apks
```

**Q: Какая минимальная версия Android для AAB?**
A: minSdkVersion 21 (Android 5.0), но лучше 26+.
