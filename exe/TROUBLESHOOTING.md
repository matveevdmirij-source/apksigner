## 🛠️ Ошибки инструментов

### Проблема: JDK не установлен

**Симптомы:**
```
'java' is not recognized as an internal or external command
'javac' is not recognized
'keytool' is not recognized
'jarsigner' is not recognized
```

**Причины:**
- JDK никогда не был установлен на компьютер
- Установлена только JRE (Java Runtime Environment)
- Путь к JDK не добавлен в переменную PATH
- JDK установлена, но в системе используется старая версия

**Пошаговое решение:**

1. **Проверить что установлено:**
   ```bash
   java -version
   javac -version
   ```
   - Если обе команды работают - JDK установлена ✅
   - Если не работают - перейти к шагу 2

2. **Скачать и установить JDK:**
   - Перейти на https://www.oracle.com/java/technologies/downloads/
   - Скачать JDK (не JRE!) для Windows
   - Запустить установщик
   - При установке **обязательно** выбрать "Add to PATH"
   - Завершить установку

3. **Проверить установку:**
   - Перезагрузить компьютер
   - Открыть новое окно Command Prompt
   - Выполнить:
   ```bash
   java -version
   javac -version
   ```

4. **Если всё ещё не работает:**
   - Добавить в PATH вручную:
     1. Нажать Win+X → Параметры системы
     2. Перейти в "Дополнительные параметры системы"
     3. Нажать "Переменные среды"
     4. Добавить новую переменную JAVA_HOME:
        - Имя: JAVA_HOME
        - Значение: C:\Program Files\Java\jdk-[версия]
     5. Добавить к PATH:
        - %JAVA_HOME%\bin
     6. Нажать OK, перезагрузить компьютер

### Проблема: Версия Java слишком старая

**Симптомы:**
```
Unsupported major.minor version
ClassFormatError: version mismatch
```

**Причины:**
- Установлена Java 7 или ранее
- Программа требует Java 8 или выше
- В PATH добавлена старая версия Java

**Решение:**

```bash
# Проверить версию:
java -version

# Должно быть: java version "11" или выше

# Если внизу, обновить:
# 1. Скачать новую версию JDK
# 2. Установить в новую папку (не перезаписывать)
# 3. Обновить JAVA_HOME на новый путь
# 4. Перезагрузить компьютер
```

## 📁 Ошибки файлов

### Проблема: Keystore повреждён или утерян

**Симптомы:**
```
Keystore was tampered with, or password was incorrect
Ошибка: java.io.IOException: Invalid keystore format
Unable to read keystore
```

**Причины:**
- Файл был удалён или переименован
- Файл поврежден при копировании
- Антивирус заблокировал или повредил файл
- Была неправильная операция шифрования
- Пароль был изменён

**Диагностика:**

```bash
# 1. Проверить существование файла:
ls -la app-release-key.jks    # Linux/Mac
dir app-release-key.jks        # Windows

# 2. Проверить целостность:
file app-release-key.jks       # Linux/Mac
# Должно выводить что-то типа: "Java KeyStore"

# 3. Попытаться открыть:
keytool -list -keystore app-release-key.jks -storepass android_key_password

# 4. Если ошибка - файл повреждён
```

**Восстановление:**

```bash
# ВАЖНО: Если у вас есть резервная копия:
copy backup\app-release-key.jks .

# Если резервной копии нет - создать новый keystore:
# 1. Удалить повреждённый файл
# 2. Запустить программу - она создаст новый keystore
# 3. ВНИМАНИЕ: Потеряются старые подписи!

# Если нужно обновлять приложение:
# - Нужно использовать ТОТ ЖЕ keystore
# - Без него невозможно обновить приложение на маркетах
# - Это серьёзная проблема!
```

### Проблема: APK файл повреждён

**Симптомы:**
```
jarsigner: unable to sign jar
jar not signed: ...
Cannot read file
Invalid archive
```

**Причины:**
- APK файл неполноценно загружен из интернета
- Файл повреждён при копировании
- Компилятор создал неправильный APK
- Антивирус повредил файл

**Проверка:**

```bash
# Метод 1: Попробовать открыть архиватором
# - Скачать 7-Zip (https://www.7-zip.org/)
# - Правый клик на APK → 7-Zip → Открыть архив
# - Если открывается → файл в порядке ✅
# - Если ошибка → файл повреждён ❌

# Метод 2: Проверить размер файла
# файл должен быть > 1 MB (обычно 20-100 MB)
# Если размер < 100 KB → вероятно повреждён

# Метод 3: Проверить хеш (если скачан с интернета)
# На сайте должен быть хеш: MD5, SHA1 или SHA256
certutil -hashfile app-release.apk SHA256  # Windows
shasum -a 256 app-release.apk               # Linux/Mac
```

**Решение:**

```bash
# 1. Если файл скачан:
# - Скачать заново
# - Проверить хеш

# 2. Если файл из IDE (Android Studio):
# - Перестроить APK (Rebuild Project)
# - Убедиться, что нет ошибок компиляции

# 3. Если файл скопирован:
# - Скопировать заново медленнее (без спешки)
# - Проверить целостность после копирования
```

### Проблема: Неправильный путь к файлам

**Симптомы:**
```
File not found
Path does not exist
No such file or directory
```

**Причины:**
- Опечатка в пути
- Файл был переназван или удален
- Используется относительный путь вместо абсолютного
- Путь содержит спецсимволы

**Решение:**

```bash
# 1. Использовать абсолютный путь:
# Вместо: app.apk
# Использовать: C:\Users\Admin\Desktop\exe\app.apk

# 2. Проверить путь:
# Скопировать полный путь из адресной строки файлового менеджера

# 3. Избегать спецсимволов:
# Хорошо:  app-release.apk
# Плохо:   app@release.apk, app#1.apk, app(final).apk

# 4. Избегать пробелов и кириллицы:
# Хорошо:  C:\signed_apps\app.apk
# Плохо:   C:\My Documents\app.apk
#          C:\Документы\приложение.apk
```

## 🔐 Ошибки безопасности

### Проблема: Неправильный пароль

**Симптомы:**
```
Keystore password was incorrect
Private key password is incorrect
Password verification failed
```

**Причины:**
- Опечатка при вводе пароля
- Пароли ключа и keystore отличаются
- Пароль содержит спецсимволы
- Забыли оригинальный пароль

**Стандартные пароли в этой программе:**
```
Пароль keystore: android_key_password
Пароль ключа:    android_key_password
Оба одинаковые!
```

**Решение:**

```bash
# 1. Проверить пароль для существующего keystore:
keytool -list -keystore app-release-key.jks -storepass android_key_password

# 2. Если команда работает - пароль правильный

# 3. Если неправильный пароль:
# - Опять создать новый keystore
# - Использовать стандартный пароль

# 4. Если забыли пароль:
# - Единственное решение: удалить keystore и создать новый
# ВНИМАНИЕ: Потеряется возможность обновлять приложение!
```

### Проблема: Отсутствуют права доступа

**Симптомы:**
```
Permission denied
Access is denied
file is read-only
Cannot write file
```

**Причины:**
- Файл отмечен как "только для чтения"
- Папка защищена (Program Files, Windows System32)
- Недостаточно прав администратора
- Файл используется другой программой

**Решение:**

```bash
# 1. Запустить от администратора:
# Правый клик на run_java.bat → "Запустить от администратора"

# 2. Снять атрибут "только для чтения":
# Правый клик на файл → Свойства
# Убрать галочку "Только для чтения"
# Нажать OK

# 3. Проверить папку:
# Папка должна быть в пользовательской директории
# Хорошо:   C:\Users\Admin\Desktop\
# Плохо:    C:\Program Files\
#           C:\Windows\
#           C:\ProgramData\

# 4. Переместить в безопасную папку:
mkdir C:\signed_apps
copy app.apk C:\signed_apps\
# Работать из папки C:\signed_apps

# 5. Закрыть другие приложения:
# IDE (Android Studio, Visual Studio)
# Архиваторы (7-Zip, WinRAR)
# Антивирусы
```

## 💾 Ошибки памяти и производительности

### Проблема: Недостаточно оперативной памяти

**Симптомы:**
```
OutOfMemoryError: Java heap space
java.lang.OutOfMemoryError
Can't allocate memory
```

**Причины:**
- APK файл очень большой (500+ MB)
- На компьютере мало свободной памяти
- Java не был выделен достаточно памяти
- Утечка памяти в программе

**Решение:**

```bash
# 1. Проверить свободную память:
# Windows: Ctrl+Shift+Esc → Память (должно быть > 2 GB)
# Linux: free -m

# 2. Закрыть ненужные приложения:
# Браузеры (Firefox, Chrome занимают 300-500 MB каждый)
# IDE и редакторы
# Потоковые сервисы

# 3. Увеличить heap size для Java (для Java версии):
# Вместо:
java APKSigner

# Использовать:
java -Xmx2048m APKSigner    # 2 GB памяти
java -Xmx4096m APKSigner    # 4 GB памяти (если есть)

# 4. Увеличить виртуальную память Windows:
# Param.csv → Система → Доп. параметры → Производительность
# Виртуальная память → Изменить
```

### Проблема: Процесс зависает / не отвечает

**Симптомы:**
```
Программа не отвечает
Завис на 50% процесса
Подпись длится очень долго
```

**Причины:**
- Очень большой APK файл
- Недостаточно памяти
- Проблемы с диском (медленный HDD)
- Конфликт с антивирусом

**Решение:**

```bash
# 1. Дождаться завершения (может занять несколько минут)

# 2. Если не помогает - закрыть программу:
# Ctrl+C в Command Prompt
# Или: Диспетчер задач → java → Завершить

# 3. Проверить место на диске:
# Windows: Щелчок на диск C: → Свойства
# Должно быть минимум 10 GB свободно

# 4. Использовать SSD вместо HDD:
# SSD работает гораздо быстрее
# Скопировать файлы на SSD диск

# 5. Отключить антивирус:
# Временно (только для тестирования!)
# Антивирусы часто замедляют работу файловых операций
```

## 🔤 Ошибки кодирования

### Проблема: Кириллица (русские буквы) в пути

**Симптомы:**
```
UnicodeDecodeError
UnicodeEncodeError
Ошибка кодирования символов
Неправильные символы в выводе
```

**Причины:**
- Путь содержит русские буквы (Па←тиОбработка)
- Команда содержит кириллицу
- Конфликт между UTF-8 и Windows-1251

**Решение:**

```bash
# 1. Переместить файлы в папку с латиницей:
mkdir C:\signed_apps
move app.apk C:\signed_apps\
move certificate.pem C:\signed_apps\

# 2. Избегать папок с русскими названиями:
# Плохо:
# C:\Пользователи\Админ\Рабочий стол\
# C:\Документы\Приложения\

# Хорошо:
# C:\Users\Admin\Desktop\
# C:\signed_apps\
# C:\work\

# 3. Переименовать папки:
# Рабочий стол → Desktop
# Мои документы → Documents
# Загрузки → Downloads

# 4. Установить UTF-8 в консоль Windows:
chcp 65001

# 5. После этого запустить программу:
python main.py
# или
java APKSigner
```

## 🔧 Инструменты диагностики

### Команда 1: Полная проверка Java

```bash
java -version
javac -version
echo %JAVA_HOME%
echo %PATH%
keytool -help
jarsigner -help
```

**Результат:** Должны вывести информацию, а не ошибки

### Команда 2: Проверка keystore

```bash
keytool -list -keystore app-release-key.jks -storepass android_key_password
```

**Результат:**
```
Keystore type: JKS
Keystore provider: SUN

Your keystore contains 1 entry

release_key, [key entry]
Certificate fingerprint (SHA1): XX:XX:XX:...
```

### Команда 3: Проверка подписанного APK

```bash
jarsigner -verify app-signed.apk
jarsigner -verify -verbose app-signed.apk
```

**Результат должен содержать:**
```
jar verified.
```

### Команда 4: Проверка места на диске

```bash
# Windows
dir C:\             # Показать место на диске C:

# Linux/Mac
df -h               # Показать место на всех дисках
```

### Команда 5: Проверка APK файла

```bash
# Windows
certutil -hashfile app-release.apk SHA256

# Linux/Mac
shasum -a 256 app-release.apk

# Результат: должен совпадать с хешем на сайте (если скачан)
```

## 🚑 Пошаговое восстановление

### Сценарий 1: Программа выдаёт неизвестную ошибку

**Шаги:**

1. Записать полный текст ошибки
2. Проверить логи в окне программы
3. Выполнить:
   ```bash
   check_requirements.bat   # Для Python версии
   # или просто запустить программу и посмотреть вывод
   ```
4. Убедиться, что:
   - ✅ Java установлена и в PATH
   - ✅ Python установлен (для Python версии)
   - ✅ Свободно >= 2 GB памяти
   - ✅ Место на диске >= 10 GB

5. Перезагрузить компьютер
6. Попробовать заново

### Сценарий 2: Потерян keystore файл

**Шаги:**

1. Попытаться найти файл:
   ```bash
   dir /s app-release-key.jks  # Windows
   find . -name "app-release-key.jks"  # Linux/Mac
   ```

2. Если найден - восстановить:
   ```bash
   # Скопировать в рабочую папку
   copy найденный_путь\app-release-key.jks .
   ```

3. Если не найден - **ПРОБЛЕМА**:
   - ❌ Больше не сможете обновлять приложение на маркетах
   - ❌ Нельзя использовать старый keystore
   - ✅ Только решение: создать новое приложение с новым keystore
   - ✅ Старое приложение останется на маркетах, но обновить не сможете

### Сценарий 3: APK не подписывается

**Шаги:**

1. Проверить APK:
   ```bash
   # Открыть 7-Zip и проверить содержимое
   # Или: certutil -hashfile app.apk SHA256
   ```

2. Если файл нормально открывается:
   ```bash
   # Проверить keystore
   keytool -list -keystore app-release-key.jks -storepass android_key_password
   ```

3. Если keystore в порядке:
   ```bash
   # Попробовать подписать вручную
   jarsigner -keystore app-release-key.jks -storepass android_key_password \
   -keypass android_key_password -signedjar app-signed.apk app.apk release_key
   ```

4. Если выдаёт ошибку - проанализировать её

### Сценарий 4: Компьютер медленный и зависает

**Шаги:**

1. Закрыть все приложения кроме необходимого

2. Проверить место на диске:
   ```bash
   dir C:\      # Windows
   df -h        # Linux/Mac
   ```
   Должно быть минимум 10 GB

3. Если диск переполнен:
   ```bash
   # Удалить ненужные файлы
   # Очистить Корзину
   # Запустить Очистку диска
   ```

4. Использовать SSD вместо HDD

5. Увеличить virtual memory (если используется Java):
   ```bash
   java -Xmx2048m APKSigner
   ```

## 📞 Когда просить помощь

Если ничего не помогло, соберите информацию:

1. **Окружение:**
   ```bash
   java -version
   javac -version
   python --version    (если используется Python)
   ```

2. **Логи:**
   Скопируйте весь вывод из окна программы/консоли

3. **Информация о файлах:**
   ```bash
   dir app.apk
   dir app-release-key.jks
   ```

4. **Информация о компьютере:**
   - ОС: Windows 10/11, macOS, Linux (какой дистрибьютив)
   - Память: сколько GB?
   - Место на диске: сколько свободно?

5. **Точное описание ошибки:**
   - Когда появляется (при выборе файла, при подписи)?
   - Какой текст ошибки?
   - Что вы делали перед ошибкой?
