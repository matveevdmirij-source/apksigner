# 🚀 Быстрый старт за 5 минут

## 🆕 Новое в v1.2: Поддержка AAB!

Приложение теперь поддерживает оба основных формата:
- **APK** - для всех маркетов (Google Play, RuStore)

**Нужен AAB?** → Перейти на [QUICK_AAB_START.md](QUICK_AAB_START.md) для быстрого старта с AAB

## За 5 простых шагов к подписанному APK!

### ✅ Шаг 1: Установить требования (одноразово)

Запустить в Command Prompt:

```bash
check_requirements.bat
```

Если всё зелёное - переходите к шагу 2!

**Если что-то не прошло:** смотрите `INSTALL.md`

---

### ✅ Шаг 2: Подготовить APK

1. Скопируйте ваш APK файл в папку с программой
   ```
   C:\Users\Admin\Desktop\exe\
   ```

2. Убедитесь, что файл называется правильно (например: `app-release.apk`)

---

### ✅ Шаг 3: Выбрать способ подписи

#### Вариант A: Просто и удобно (Графический интерфейс)

```bash
run.bat
```

**Что делать:**

1. Нажмите "Обзор" рядом с "Выбор APK файла"
2. Выберите ваш APK
3. Нажмите "Обзор" рядом с "Папка для сохранения файлов"
4. Выберите папку (например: `C:\signed_apps`)
5. Нажмите **"Генерировать ключи и подписи"**
6. Ждите завершения (смотрите лог)

**Четыре файла готовы! ✅**

---

#### Вариант B: Быстро из командной строки

```bash
python cli.py app-release.apk -o output_folder
```

**Четыре файла готовы! ✅**

---

### ✅ Шаг 4: Получить файлы

В папке `output_folder` будут:

- `app-signed.apk` - 📱 **Ваше подписанное приложение**
- `certificate.pem` - 🔐 **Сертификат**
- `playmarket_info.txt` - 📊 **Инфа для Play Market**
- `rustore_info.txt` - 📊 **Инфа для RuStore**

---

### ✅ Шаг 5: Загрузить на маркет

#### Для Google Play Market:

1. Откройте https://play.google.com/console
2. Создайте новое приложение
3. Загрузите `app-signed.apk`
4. Скопируйте SHA256 из `playmarket_info.txt`
5. Вставьте в систему Play

#### Для RuStore:

1. Откройте https://developer.rustore.ru/
2. Создайте новое приложение
3. Загрузите `app-signed.apk`
4. Загрузите `certificate.pem`
5. Проверьте SHA256 совпадение

## 🎯 Самые частые вопросы

### ❓ Что такое Keystore?
**Ответ:** Это ваш "ключ" для подписи приложения. Его создаёт программа автоматически.

### ❓ Неужели так просто?
**Ответ:** Да! Всё остальное программа делает автоматически.

### ❓ Что если нужно обновить приложение?
**Ответ:** Используйте тот же keystore (программа его сохранит). 

**Команда:**
```bash
python cli.py app-release-v2.apk -k output_folder\app-release-key.jks
```

### ❓ Что если я потеряю keystore?
**Ответ:** 😭 Вы не сможете обновить приложение. **Сохраняйте его!**

### ❓ Где найти SHA256?
**Ответ:** В файле `playmarket_info.txt` или `rustore_info.txt`

### ❓ Работает на Mac?
**Ответ:** Да! Используйте:
```bash
python3 main.py      # Графический интерфейс
python3 cli.py app.apk -o output   # Командная строка
```

### ❓ Работает на Linux?
**Ответ:** Да! Используйте:
```bash
python3 main.py      # Графический интерфейс
python3 cli.py app.apk -o output   # Командная строка
```

## 🚨 Если что-то не работает

### Ошибка #1: "Python not found"
```
✅ Решение: Переустановить Python
Скачать: https://www.python.org/downloads/
```

### Ошибка #2: "Java not found"
```
✅ Решение: Переустановить Java JDK
Скачать: https://www.oracle.com/java/technologies/downloads/
```

### Ошибка #3: "APK не найден"
```
✅ Решение: Скопировать APK в папку программы
Или: Указать полный путь к файлу
```

### Ошибка #4: "Разрешение запрещено"
```
✅ Решение: Запустить Command Prompt от администратора
Или: Переместить папку в документы
```

**Если всё ещё не работает:**
1. Запустите `check_requirements.bat`
2. Прочитайте `INSTALL.md`
3. Посмотрите `EXAMPLES.md` для подробных примеров

## 💡 Полезные советы

✅ **Совет 1:** Используйте одну папку для всех проектов
```
C:\signed_apps\app1\
C:\signed_apps\app2\
```

✅ **Совет 2:** Сохраняйте keystore в безопасном месте
```
C:\backup\keystore\app-key.jks
```

✅ **Совет 3:** Проверяйте SHA256 всегда!
```
Без совпадения SHA256 маркет может не распознать обновление
```

✅ **Совет 4:** Создавайте опись
```
Сохраняйте файлы playmarket_info.txt и rustore_info.txt
для документирования
```
