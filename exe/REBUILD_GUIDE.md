# 🔧 APK Signer Advanced - Пересборка Debug в Release (v1.2+)

Расширенная версия приложения с функцией пересборки Android проекта из debug режима в release режим.

## 📋 Новые возможности

✅ **Поддержка APK и AAB** - Выбирайте формат для подписи  
✅ **Сборка Release версии** - Использует Gradle для пересборки APK в режиме Release  
✅ **Автоматическая подпись** - Подписывает собранный Release APK нашим ключом  
✅ **Поддержка gradlew** - Автоматически использует gradlew если он доступен  
✅ **Две вкладки** - "Подпись APK/AAB" и "Пересборка Release"  
✅ **Детальное логирование** - Полный вывод процесса пересборки  

## ⚠️ Важно о форматах

### Вкладка "Подпись APK/AAB":
- Поддерживает ОБА формата: **APK и AAB**
- **APK**: для RuStore и локального тестирования
- **AAB**: только для Google Play Store

### Вкладка "Пересборка Release":
- Создает только **APK** формат (не AAB!)
- Причина: Gradle собирает APK, AAB требует дополнительного инструмента bundletool
- После пересборки APK вы можете:
  - ✅ Загрузить на RuStore как есть
  - ✅ Загрузить на Google Play (система конвертирует в AAB)
  - ✅ Подписать еще раз через вкладку "Подпись" если нужно

> **Для AAB на Google Play:** Используйте готовый AAB файл на вкладке "Подпись APK/AAB"

## 📖 Два Режима Работы

### Режим 1: Подпись готового APK/AAB (Вкладка "Подпись APK/AAB")

Для когда у вас уже готов скомпилированный файл:

1. **Выбрать формат** - APK или AAB
2. **Выбрать файл** - диалог будет фильтровать по выбранному формату
3. **Выбрать/создать Keystore**
4. **Выбрать папку вывода**
5. **Нажать "Генерировать ключи и подписи"**

Результат:
- `app-signed.apk` (если выбран APK формат)
- `app-signed.aab` (если выбран AAB формат)
- Сертификаты и инструкции по загрузке

### Режим 2: Пересборка Android проекта (Вкладка "Пересборка Release")

Для когда у вас есть исходный код Android проекта:

1. **Выбрать папку проекта Android**
   - Это должна быть корневая папка проекта (где находится build.gradle)
   - Проект должен иметь структуру:
   ```
   MyProject/
   ├── build.gradle
   ├── settings.gradle
   ├── gradlew.bat / gradlew
   └── app/
       ├── build.gradle
       ├── src/
       └── ...
   ```

2. **Выбрать/создать Keystore файл (опционально)**
   - Если не выбрать - будет создан новый keystore в папке проекта

3. **Нажать "Пересобрать и подписать Release"**

Программа выполнит:
- ✅ Очистку предыдущих сборок (gradle clean)
- ✅ Сборку Release версии (gradle assembleRelease)
- ✅ Поиск сгенерированного APK
- ✅ Создание/использование Keystore
- ✅ Подпись APK нашим ключом
- ✅ Экспорт сертификата в PEM
- ✅ Генерацию информации для маркетов

## 📊 Процесс пересборки

### Этап 1: Готовка
```
✓ Анализ проекта Android
✓ Проверка наличия build.gradle
✓ Проверка наличия gradlew
```

### Этап 2: Очистка
```
gradle clean
- Удаляет все предыдущие сборки
- Очищает директорию build/
```

### Этап 3: Сборка Release
```
gradle assembleRelease
- Компилирует исходный код
- Оптимизирует ресурсы
- Создаёт Release версию APK (без подписи)
- Сохраняет в: app/build/outputs/apk/release/
```

### Этап 4: Подпись Release
```
jarsigner
- Подписывает Release APK нашим ключом
- Проверяет целостность
- Сохраняет в: project/release_signed/
```

### Этап 5: Экспорт сертификатов
```
keytool -export
- Экспортирует публичный сертификат
- Вычисляет отпечатки SHA256 и SHA1
- Создаёт информацию для маркетов
```

## 🛠️ Возможные проблемы при пересборке

### Проблема 1: "Gradle не найден"

**Решение:**
```bash
# Установить gradle
# Windows через Chocolatey:
choco install gradle

# Или скачать вручную:
# https://gradle.org/releases/

# Или использовать gradlew из проекта
```

### Проблема 2: "Cannot find symbol" при компиляции

**Решение:**
```bash
# Обновить зависимости:
# 1. Открыть build.gradle
# 2. Обновить версии compileSdkVersion, targetSdkVersion
# 3. Запустить: gradle clean build

# Или полностью пересобрать в Android Studio
```

### Проблема 3: "Compile SDK version is not set"

**Решение:**
```bash
# 1. Открыть app/build.gradle
# 2. Найти строку: compileSdkVersion
# 3. Установить версию (например: compileSdkVersion 34)
# 4. Повторить пересборку
```

### Проблема 4: "Gradle build failed"

**Решение:**
```bash
# 1. Проверить логи (они выводятся в окне программы)
# 2. Открыть проект в Android Studio
# 3. Выполнить Build → Clean Project
# 4. Выполнить Build → Rebuild Project
# 5. Если работает в Android Studio - пересборка через программу тоже должна работать
```

### Проблема 5: "APK not found"

**Решение:**
```bash
# Проверить структуру проекта:
# app/build/outputs/apk/release/app-release.apk должен существовать

# Если нет:
# 1. Проверить логи gradle
# 2. Убедиться, что gradleassembleRelease завершился без ошибок
# 3. Проверить место на диске (нужно > 5 GB)
```

## 🔐 Важные замечания о безопасности

⚠️ **КРИТИЧНО:**
1. **Используйте ОДИ Keystore для версии** - не создавайте новый для каждой сборки!
2. **Сохраняйте Keystore в безопасном месте** - без него не сможете обновлять приложение
3. **Никогда не теряйте пароль** - стандартный: android_key_password
4. **Не публикуйте keystore** файл нигде!
