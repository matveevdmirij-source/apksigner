✨ ЧТО НОВОГО: AAB ПОДДЕРЖКА

Теперь ты можешь подписывать оба формата:

  APK - Традиционный формат (как раньше)
    ✅ Google Play Store
    ✅ RuStore
    ✅ Все маркеты
    ✅ Размер: 40-100 MB
  
  AAB - Новый формат (добавлено в v1.2!) 🆕
    ✅ Только Google Play Store
    ✅ На 40% меньше размер
    ✅ Автоматическая оптимизация для девайса
    ✅ Размер: 10-30 MB

📖 БЫСТРЫЙ СТАРТ

За 5 минут до подписанного AAB:

  1. Создать AAB в Android Studio:
     Build → Build Bundle(s) / APK(s) → Build Bundle(s)
  
  2. Запустить APK Signer:
     run_java.bat
  
  3. В окне программы:
     а) Выбрать формат: AAB (из выпадающего списка)
     б) Выбрать файл: ваш создал app-release.aab
     в) Нажать: "Генерировать ключи и подписи"
  
  4. Получить результат:
     ✅ app-signed.aab (подписанный файл для Google Play)
     ✅ certificate.pem (сертификат)
     ✅ playmarket_info.txt (инструкции для Google Play)
  
  5. Загрузить на Google Play:
     Google Play Console → Release → Upload app-signed.aab

Подробная инструкция в файле: QUICK_AAB_START.md

⚠️ ВАЖНО: ПРАВИЛА

1. ИСПОЛЬЗУЙ ОДИН KEYSTORE ДЛЯ ВСЕГО ПРИЛОЖЕНИЯ!
   
   ✅ Правильно:
      Keystore A → app v1.0 APK + app v1.0 AAB + app v1.1 APK + app v1.1 AAB
   
   ❌ Неправильно (не делай так!):
      app v1.0 APK → Keystore A
      app v1.0 AAB → Keystore B (разные keystore = ошибка!)

2. ЗАЩИЩАЙ KEYSTORE!
   
   ✅ Сохрани в безопасном месте
   ✅ Запомни пароль: android_key_password
   ✅ Сделай резервную копию
   ❌ НЕ выкладывай на GitHub/интернет

3. ОДИНАКОВЫЙ KEYSTORE ДЛЯ ОБНОВЛЕНИЙ!
   
   Когда обновляешь приложение:
   - Используй ТОТ ЖЕ Keystore что раньше
   - Никаких новых ключей не создавай
   - Google Play будет знать что это обновление
