## 📋 Что изменилось?

### Java версии (APKSigner.java, APKSignerAdvanced.java)

#### Новые компоненты:
```
✅ Format selector (JComboBox)
   - Позволяет выбрать APK или AAB

✅ Dynamic file filtering
   - Диалог фильтрует файлы по выбранному формату

✅ Format-specific signing
   - APK: подписывается через jarsigner (как ранее)
   - AAB: подписывается той же системой, но без RuStore поддержки

✅ Smart instructions
   - Разные инструкции для APK и AAB
   - AAB показывает только Google Play информацию
```

#### Измененные методы в APKSigner.java:

- analyzeAppFile(path, format) // Анализирует оба формата

// Обновленные методы:

## 🔄 Процесс подписи

### Для APK (как раньше):
```
1. Выбрать APK файл
2. Создать/выбрать Keystore
3. Подписать jarsigner
4. Экспортировать сертификат
5. Сгенерировать информацию для Play Market и RuStore
```

### Для AAB (новое):
```
1. Выбрать AAB файл
2. Создать/выбрать Keystore (ОДИНАКОВЫЙ для APK и AAB!)
3. Подписать jarsigner
4. Экспортировать сертификат
5. Сгенерировать информацию для Google Play Store только
   (RuStore не поддерживает AAB)
```

## 🚀 Как начать использовать?

### Квиккстарт: AAB на Google Play

1. **Скачать AAB**
   ```
   Android Studio → Build → Build Bundle(s) / APK(s) → Build Bundle(s)
   ```

2. **Запустить подписщик**
   ```bash
   run_java.bat        # или
   run_advanced.bat
   ```

3. **Выбрать формат: AAB**
   - Из выпадающего списка (новое!)

4. **Выбрать файл**
   - Диалог будет показывать только *.aab файлы

5. **Подписать**
   - Нажать "Генерировать ключи и подписи"

6. **Загрузить на Google Play**
   ```
   Google Play Console → Release → Upload app-signed.aab
   ```

## 🔐 Безопасность

### Ничего не изменилось:
✅ Алгоритм подписи: RSA 2048-bit, SHA1  
✅ Хранилище ключей: Keystore  
✅ Пароли: Те же (android_key_password)  
✅ ОДИН Keystore на версию приложения (правило не изменилось!)  

### Новые рекомендации:
⚠️ **ВАЖНО:** Используйте ОДИНАКОВЫЙ Keystore для APK и AAB!
```
Правильно:
- Keystore v1: используется для app v1.0 APK и app v1.0 AAB

Неправильно (не делайте так!):
- APK podpisano s Keystore A
- AAB podpisano s Keystore B
→ Google Play отклонит обновление
```

## ⚠️ Известные ограничения

1. **Python версии (пока не обновлены)**
   - main.py не имеет GUI для выбора формата
   - cli.py не имеет --format аргумента
   - Планируется обновить в ближайшее время

2. **RuStore + AAB**
   - RuStore не поддерживает AAB формат
   - Можно использовать только APK для RuStore

3. **bundletool**
   - Для конвертирования AAB → APK (локальное тестирование) нужен bundletool
   - Этот инструмент НЕ входит в состав приложения
   - Требуется установить отдельно если нужно

4. **Android версия**
   - minSdkVersion рекомендуется 21+
   - targetSdkVersion 34+
